# Worth Then

책 속에 등장하는 옛돈이 오늘날 어느 정도의 가치인지 빠르게 가늠하기 위한 작은 웹 계산기입니다.

아가사 크리스티 같은 근현대 소설을 읽다 보면 "10파운드", "500프랑" 같은 금액이 자주 등장하지만, 당시 독자에게는 자연스러웠던 체감 가치를 지금의 원화나 달러로 바로 떠올리기 어렵습니다. 이 프로젝트는 책을 읽는 흐름을 끊지 않고 대략적인 현재 가치로 환산해 보는 것을 목표로 합니다.

이 앱은 정교한 회계, 투자, 법률 판단을 위한 환산기가 아닙니다. 목적은 독서 중 시대상과 금액 감각을 이해하기 위한 참고값을 제공하는 것입니다.

## 목표

- 정적 HTML, CSS, JavaScript만으로 동작하는 단순한 웹 앱을 만든다.
- GitHub Pages에 배포해 모바일 브라우저에서 바로 사용할 수 있게 한다.
- 사용자가 옛 화폐, 금액, 기준 연도, 목표 연도, 목표 화폐를 선택하면 대략적인 현재 가치를 보여준다.
- 계산 결과가 "정확한 환전 금액"이 아니라 "독서용 참고값"임을 분명히 표시한다.
- 데이터는 개발 시점에 미리 수집해 정적 파일로 저장하고, 브라우저에서는 외부 API를 실시간 호출하지 않는다.

## 초기 기능 범위

### 입력

- 대상 화폐
  - 영국 파운드
  - 프랑스 프랑
  - 유럽 유로
  - 미국 달러
  - 제정 러시아 루블 (1880-1913, 실험적 추정)
  - 소련 루블 (1961-1991, 실험적 추정)
  - 러시아 구루블 (1992-1997)
  - 러시아 루블 (1998년 이후)
- 대상 가격
  - 숫자 입력
- 기준 연도
  - 책 속 금액이 등장하는 시대의 연도
  - 예: 1920년, 1934년, 1950년
- 목표 연도
  - 기본값: 2026년
  - 사용자가 수정 가능
- 목표 화폐
  - 원화
  - 미국 달러
  - 기본값: 원화

### 동작

- 변환 버튼을 누르면 입력한 금액을 목표 연도 기준의 목표 화폐 가치로 환산한다.
- 영국 구화폐 보조 변환기에서 파운드, 실링, 펜스를 입력하면 십진 파운드로 합산한 뒤 현재 가치로 환산한다.
- 결과 영역에는 계산된 대략적 금액과 함께 기준 정보, 근사치 안내, 데이터 출처를 짧게 보여준다.
- 2026년처럼 연간 평균 데이터가 아직 확정되지 않은 연도는 최신 데이터 또는 추정값 기반임을 표시한다.
- 모바일 화면에서도 입력과 결과 확인이 편해야 한다.

## 사전 조사 결론

초기 지원 화폐 중 영국 파운드, 프랑스 프랑, 유럽 유로, 미국 달러는 공개 데이터로 구현 가능성이 충분합니다. 독일 탈러는 시대와 지역에 따라 단위가 크게 달라 단일한 연도별 구매력 데이터로 다루기 어렵기 때문에 초기 스펙에서 제외합니다.

### 화폐별 데이터 방향

- 영국 파운드
  - Bank of England의 inflation calculator와 ONS 물가지수 자료를 기준으로 삼을 수 있습니다.
  - 장기 과거 데이터가 비교적 잘 정리되어 있어 독서용 구매력 환산에 적합합니다.
- 프랑스 프랑
  - INSEE의 franc-euro 구매력 변환 자료를 기준으로 삼을 수 있습니다.
  - 프랑, 구프랑, 유로 전환과 물가 반영 방식이 함께 제공됩니다.
- 유럽 유로
  - Eurostat HICP와 ECB 환율 자료를 조합해 사용할 수 있습니다.
  - 유로 도입 이전 국가별 통화는 각 국가의 공식 고정 전환율을 통해 다루는 방식을 우선합니다.
- 미국 달러
  - BLS CPI 자료를 1차 기준으로 사용할 수 있습니다.
  - FRED는 보조 조회나 검증용으로 활용할 수 있습니다.
- 러시아 루블
  - Historicalstatistics.org의 Historical Currency Converter에 있는 러시아 루블 장기 변환 자료를 실험적 참고값으로 사용합니다.
  - 제정 러시아 루블은 실제 표 값이 있는 1880-1913년만 지원합니다.
  - 소련 루블은 1961년 화폐개혁 이후부터 소련 붕괴 전인 1991년까지 별도 선택지로 다룹니다.
  - 이 장기 변환 자료는 스웨덴 소비재 구매력 비교를 통해 목표 연도 미국 달러 감각으로 이어 붙이는 방식이라 정확도가 낮을 수 있습니다.
  - 러시아 연방 구간은 World Bank의 러시아 CPI와 공식 환율 자료를 기준으로 삼을 수 있습니다.
  - 1998년 화폐개혁 전 금액은 `RUR`, 1998년 이후 금액은 `RUB`로 나눠 다룹니다.
  - 1998년 전환율 `1 RUB = 1,000 RUR`을 명시적으로 반영합니다.
  - 1914-1960 구간은 자료 공백과 1920년대/1947년 화폐개혁 해석 문제 때문에 현재 범위에서 제외합니다.
- 원화
  - 초기에는 목표 화폐로만 사용합니다.
  - 목표 연도의 USD/KRW 또는 EUR/KRW 평균 환율을 사용해 최종 원화 금액을 계산합니다.
  - 필요해지면 한국 CPI를 추가해 "한국 내 구매력" 기준 결과도 별도 모드로 확장할 수 있습니다.

### 검토한 공개 데이터/API

- Bank of England inflation calculator
- Office for National Statistics CPI/RPI 계열 데이터
- INSEE franc-euro purchasing power converter
- BLS Public Data API
- Eurostat API
- ECB reference exchange rates
- World Bank Indicators API
- FRED economic data
- KOSIS OpenAPI

런타임에서 사용자의 브라우저가 이 API들을 직접 호출하지는 않습니다. API는 데이터 갱신 작업에서만 사용하고, 앱은 저장소에 포함된 정적 데이터 파일을 읽습니다.

## 계산 방식 초안

기본 계산은 "당시 현지 구매력"을 먼저 맞춘 뒤, 목표 화폐로 환산하는 방식으로 진행합니다.

```text
입력 금액
  x (대상 화폐 국가의 목표 연도 물가지수 / 대상 화폐 국가의 기준 연도 물가지수)
  = 목표 연도 기준 대상 화폐 구매력

목표 연도 기준 대상 화폐 구매력
  x 목표 연도 평균 환율
  = 목표 화폐 금액
```

예를 들어 `1930년의 10파운드`를 `2026년 원화`로 변환할 때는 다음 흐름을 따릅니다.

1. 1930년 영국 파운드 10파운드의 영국 내 구매력을 2026년 영국 파운드 가치로 환산한다.
2. 2026년 평균 GBP/KRW 환율 또는 GBP/USD와 USD/KRW 환율을 사용해 원화로 변환한다.
3. 결과에 사용한 기준 연도, 목표 연도, 물가지수 출처, 환율 출처, 추정 여부를 표시한다.

이 방식은 당시 환율만 단순 적용하는 것보다 책 속 금액의 체감 규모를 이해하는 데 더 적합합니다.

## 데이터 저장 및 갱신 설계

초기 버전은 외부 API를 실시간 호출하지 않고, 정리된 기준 데이터를 저장소에 포함해 정적으로 계산합니다.

예상 데이터 흐름:

1. 공개 API 또는 공개 파일에서 연도별 CPI, 구매력 지수, 평균 환율 데이터를 조사한다.
2. 수집 원본 또는 변환 결과를 `data/` 아래에 JSON 파일로 저장한다.
3. 데이터별 출처, 기준 단위, 마지막 갱신일, 변환 방식을 메타데이터로 남긴다.
4. 앱은 `src/`의 계산 로직에서 정적 데이터만 읽어 환산한다.
5. 추후 Codex로 데이터 갱신 작업을 실행할 수 있도록 수집 스크립트나 갱신 절차를 문서화한다.

현재 파일 구조:

```text
/
  index.html
  README.md
  AGENTS.md
  data/
    README.md
    sources.json
    price-indexes.json
    exchange-rates.json
  scripts/
    update-data.mjs
  src/
    app.js
    calculator.js
    styles.css
```

`scripts/update-data.mjs`는 공식 데이터 소스를 다시 조회하고 `data/` 파일을 갱신하는 스크립트입니다. GitHub Pages에 배포되는 앱 자체는 이 스크립트를 실행하지 않습니다.

현재 생성되는 데이터 파일:

- `data/sources.json`
  - 데이터 출처, 접근 시각, 프랑-유로 공식 고정 전환율 등
- `data/price-indexes.json`
  - GBP: 1209-2026
  - FRF: 1901-2025
  - EUR: 1996-2025, 실제 유로 계산은 1999년 이후 사용
  - USD: 1913-2026
  - RUB_IMP: 1880-1913 입력, 목표 연도는 1998-2026
  - SUR: 1961-1991 입력, 목표 연도는 1998-2026
  - RUR/RUB: 1992-2025, 입력은 RUR 1992-1997, RUB 1998-2025로 구분
- `data/exchange-rates.json`
  - USD 1달러당 현지 통화 단위의 연평균 환율
  - GBP/KRW/USD: 1960-2026
  - EUR: 1999-2026
  - FRF: 1960-1998
  - RUR: 1993, 1996-1997
  - RUB: 1998-2024

데이터 갱신 명령:

```sh
node scripts/update-data.mjs
```

2026년처럼 확정 연간 평균이 아직 없는 값은 가능한 경우 최신 월 또는 연초 이후 평균으로 저장하고, 해당 연도는 `provisionalYears`에 표시합니다.

폐지되거나 통합된 화폐는 가능한 경우 공식 고정 전환율, 역사적 환율, 물가 지수, 구매력 지표를 조합해 처리합니다. 프랑처럼 유로로 전환된 화폐는 공식 고정 전환율과 INSEE 자료를 우선합니다.

## 추가하면 좋은 아이디어

- 결과에 "매우 대략적", "자료 제한 있음" 같은 신뢰도 표시
- 책 속 표현을 그대로 적어둘 수 있는 짧은 메모 입력
- 자주 쓰는 기준 연도 빠른 선택
  - 1920년대, 1930년대, 1950년대 등
- 데이터 출처와 계산 방법을 접을 수 있는 상세 영역으로 제공
- 새 화폐를 쉽게 추가할 수 있는 데이터 구조

## 개발 원칙

- 첫 화면은 계산기 자체여야 하며, 랜딩 페이지처럼 만들지 않는다.
- 필요한 정보만 보이는 조용하고 깔끔한 인터페이스를 우선한다.
- 모바일 웹 사용성을 필수 기준으로 삼는다.
- 계산 로직과 데이터는 가능한 한 분리한다.
- 모든 기준 데이터에는 출처와 산정 방식을 남긴다.
- 계산값은 과도한 정밀도를 피하고, 독서 중 참고하기 좋은 형식으로 보여준다.
- 데이터 갱신은 사람이 검토 가능한 커밋으로 남긴다.
