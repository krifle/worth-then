# AGENTS.md

## 기본 응답 언어

기본 언어는 한국어로 합니다.

## 프로젝트 목적

이 저장소는 책 속에 등장하는 옛 화폐 금액을 오늘날의 원화 또는 달러 가치로 대략 환산해 주는 정적 웹 앱입니다. 핵심 사용 장면은 모바일 브라우저에서 독서 중 빠르게 금액 감각을 확인하는 것입니다.

## 제품 방향

- 앱의 첫 화면은 바로 사용할 수 있는 계산기여야 합니다.
- 기능은 작고 명확하게 유지합니다.
- 인터페이스는 모바일 우선으로 설계합니다.
- 설명은 필요한 만큼만 노출하고, 결과에는 계산 근거와 불확실성을 짧게 표시합니다.
- 결과는 참고용 추정치이며 정확한 환전, 회계, 투자 목적의 값처럼 표현하지 않습니다.

## 기술 방향

- 기본 구현은 HTML, CSS, JavaScript만 사용합니다.
- GitHub Pages 배포를 염두에 두고 정적 파일만으로 동작하게 만듭니다.
- 빌드 도구, 프레임워크, 패키지 매니저는 필요성이 분명할 때만 도입합니다.
- 런타임에서 외부 API에 의존하지 않는 것을 기본값으로 합니다.
- 계산 데이터와 UI 코드는 가능한 한 분리합니다.

## 데이터 원칙

- 화폐별 기준 데이터에는 출처, 기준 연도, 계산 방법을 함께 남깁니다.
- 폐지된 화폐나 통합된 화폐는 공식 전환율, 역사적 환율, 물가 지수, 구매력 자료를 검토해 근사값으로 처리합니다.
- 데이터가 부족하거나 불확실성이 큰 화폐는 UI에 이를 표시합니다.
- 새 화폐를 추가할 때는 기존 데이터 구조를 확장하는 방식으로 작업합니다.
- 과도한 소수점 정밀도는 피하고, 사용자가 읽기 쉬운 금액으로 반올림합니다.

## 초기 지원 범위

- 대상 화폐: 영국 파운드, 프랑스 프랑, 유럽 유로, 미국 달러, 독일 탈러
- 목표 화폐: 원화, 미국 달러
- 목표 연도 기본값: 2026년

## 코드 작성 기준

- 접근성과 모바일 사용성을 기본 품질 기준으로 봅니다.
- 입력값 검증과 오류 메시지를 명확하게 처리합니다.
- 화폐 추가가 쉬운 자료 구조를 선호합니다.
- CSS는 과한 장식보다 가독성, 간격, 터치 영역, 반응형 안정성을 우선합니다.
- 불필요한 추상화나 대규모 구조 변경은 피합니다.

## 검증 기준

- 모바일 폭에서 입력, 버튼, 결과 영역이 겹치지 않아야 합니다.
- 잘못된 입력, 빈 입력, 데이터가 없는 조합을 확인합니다.
- 계산 결과에는 기준 화폐, 목표 연도, 목표 화폐가 명확히 드러나야 합니다.
- 변경 후에는 가능한 범위에서 브라우저 표시와 기본 계산 흐름을 확인합니다.
